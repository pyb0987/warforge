## 패턴 1: 구조 투영 (Structure Projection)
- **현상**: 새 테마 설계 시 기존 테마(스팀펑크)의 역할 구조를 그대로 대응시킴
- **발생**: 드루이드 초기 설계에서 "스팀펑크의 조립소 = 드루이드의 ???" 대응표 작성
- **대응**: CLAUDE.md + card-design-method.md에 안티패턴으로 명시 (commit 244d6fa)

## 패턴 2: ★2 합성 유인 부족
- **현상**: ★2 카드의 효과가 ★1×2장보다 약하거나 동등하여 합성 동기 없음
- **발생**: 여러 카드의 ★2 설계에서 수치 비교 없이 감각적 설정
- **대응**: CLAUDE.md에 "★2 합성 유인 검증 (필수)" 규칙 추가, feedback_star2_design.md 체크리스트 생성

## 패턴 3: 1회성 임계점
- **현상**: "🌳8 달성 시 1회 발동" 같은 설계가 ★ 합성 메커니즘과 충돌
- **발생**: 드루이드 카드 초기안에서 임계점 + 합성 간 비호환
- **대응**: "지속 조건부"로 전환 (🌳8 이상이면 매 라운드)
