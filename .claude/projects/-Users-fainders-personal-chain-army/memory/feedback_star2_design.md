★2 또는 ★3 카드를 작성/수정한 후 반드시 이 체크리스트를 수행한다.

## ★2 합성 유인 체크

| # | 질문 | 통과 조건 |
|---|------|-----------|
| 1 | ★1×2장 vs ★2×1장 비교 | ★2 ≥ ★1×2 출력 또는 동등+슬롯 절약 |
| 2 | 핵심 수치 ×1.5~2 범위 | 5%→7.5~10%, 1기→1.5~2기 |
| 3 | 정체성 유지 | ★1과 ★2의 트리거 조건이 동일하고, 효과 범주(제조/개량/충전 등)가 동일하면 PASS |
| 4 | 금지 위반 없음 | 상한만 +1 ✗, 범위 확대 ✗, 부가 효과 추가 ✗ |

## ★3 차원 전환 체크

| # | 질문 | 통과 조건 |
|---|------|-----------|
| 1 | 새 차원 추가 | 패시브/오라, 브릿지, 범위 확대 중 하나 이상 명시적으로 존재하면 PASS |
| 2 | ★2와 질적 차이 | ★2에 없던 새 메커니즘(조건부 발동/대상 변경/연쇄 등)이 1개 이상이면 PASS |
| 3 | 필수카드화 방지 | 해당 ★3 제거 후 테마 체인이 2개 이상 경로로 여전히 작동하면 PASS |

## 임계점 호환성 체크

| # | 질문 | 통과 조건 |
|---|------|-----------|
| 1 | 1회성 임계점 없음 | "🌳8 달성 시 1회" ✗ → "🌳8 이상이면 매 라운드" ✓ |
| 2 | ★ 합성과 비충돌 | 합성 전후 모두 효과 발동 가능 |
| 3 | 조건부 지속형 | 임계점 도달 후 지속적 가치 제공 |

## 과거 실패 패턴

- **★2 합성 유인 부족**: ★2가 ★1×2보다 약하면 합성 동기 없음 → 반드시 수치 비교
- **★2에 ★3 영역 혼입**: 범위 확대, 부가 효과를 ★2에 넣으면 ★3의 설계 공간 잠식
- **1회성 임계점 설계**: 누적/합성 메커니즘과 충돌 — 지속 조건부로 전환
