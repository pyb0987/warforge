# /btw — 사이드 질문 (컨텍스트 격리)

메인 작업 흐름을 오염시키지 않는 단발성 질문 처리.

## 동작

1. 사용자의 질문을 opus 서브에이전트에게 위임
2. 서브에이전트가 답변 (필요 시 웹 검색, 파일 읽기 가능)
3. 메인 에이전트는 답변을 사용자에게 전달만 함
4. 서브에이전트의 중간 과정은 메인 컨텍스트에 남지 않음

## 실행

```
<instructions>
사용자의 질문: {{args}}

Agent 도구로 서브에이전트를 생성하여 질문에 답변하게 한다.

- model: "opus"
- 코드베이스 참조가 필요한 질문이면 subagent_type: "Explore" 사용
- 일반 지식 질문이면 subagent_type 지정 없이 사용

프롬프트:
"다음 질문에 구조적이고 간결하게 답변해줘. 한국어로 답변. 질문: {args}"

답변을 받으면 사용자에게 그대로 전달. 추가 코멘트 불필요.
</instructions>
```
