# 렌더링 & 전투 엔진 기술 설계

## 목표 환경

- 모바일 네이티브 + 웹 (WebGL) 동시 지원
- 전투 규모: 최대 200v200 (400유닛 동시 렌더)
- 목표 프레임: 60fps, 서멀 쓰로틀링 후에도 유지

## 렌더 방식: 프레임 스프라이트

프레임 스프라이트(SC1 방식) 채택. 2D 스켈레탈 대비 400유닛에서 CPU 이점이 결정적.

| | 프레임 스프라이트 | 2D 스켈레탈 |
|:---|:---|:---|
| 애니메이션 CPU (400유닛) | ~0.1ms | 4~8ms (프레임 버짓 24~48%) |
| 서멀 쓰로틀링 내성 | 높음 | 프레임 드롭 위험 |
| 웹 메모리 | 최적화 후 40~60MB | ~15MB |
| 드로우콜 | 인스턴싱으로 1~4콜 | 배치 브레이크 잦음 |

메모리 최적화 전략:
- 해상도 차등: 소형 32×32, 중형 48×48, 대형 64×64
- 4방향 제작 + 좌우 반전 = 8방향
- 같은 유닛 타입은 텍스처 아틀라스 공유
- 모바일: ASTC/ETC2 압축 → 5~8MB
- 웹: 아틀라스 팩킹 최적화 → 40~60MB

## 유닛 비주얼 데이터

### 스프라이트 애니메이션 (per 유닛 타입)

| 상태 | 방향 | 프레임 | 용도 |
|:---|:---|:---|:---|
| Idle | symmetric: 4방향(+반전=8), asymmetric: 8방향 | 2~4 | 대기 루프 |
| Walk | symmetric: 4방향(+반전=8), asymmetric: 8방향 | 6~8 | MS 기반 재생 속도 |
| Attack | symmetric: 4방향(+반전=8), asymmetric: 8방향 | 4~6 | AS 기반 타이밍, hit_frame 포함 |
| Death | 1방향 | 4~6 | 사망 후 제거 |

### 좌우 대칭 판정 & 방향 수

```
symmetric (4방향 + 좌우 반전 = 8방향):
  대칭 디자인 유닛. 대부분의 소형/중형 유닛.
  → 스프라이트 제작량 절반.

asymmetric (8방향 전부 별도 제작):
  비대칭 디자인 유닛. 대형 유닛, 보스, 한손 무기/비대칭 장비 보유 유닛.
  → 반전 시 시각적 위화감이 있는 경우에만 적용.

판정 기준:
  - 32~64px 스프라이트에서 반전해도 인지 어려움 → symmetric
  - 명백한 비대칭(대형 유닛, 한쪽 방패/대포 등) → asymmetric
  - 유닛 DB에 symmetric 필드로 기록하여 파이프라인에서 자동 분기
```

★등급 비주얼 변화: 팔레트 스왑 (★1 기본 → ★2 강조색 → ★3 금색/발광)

### 렌더링 메타데이터

```
unit_visual = {
  id: "steam_spider",
  sprite_sheet: "steampunk/steam_spider.png",
  size_class: "small" | "medium" | "large",
  symmetric: true,              // false면 8방향 전부 별도 스프라이트
  collision_radius: float,
  shadow_size: float,

  animations: {
    idle:   { frames: [0..3],   fps: 4,  loop: true },
    walk:   { frames: [4..11],  fps: 8,  loop: true },
    attack: { frames: [12..17], fps: 6,  loop: false, hit_frame: 4 },
    death:  { frames: [18..23], fps: 6,  loop: false },
  },

  projectile: null | "bullet_brass",
  sfx: { attack, death, move },
}
```

## 유닛 스프라이트 구성 (1종 분해)

### 본체 스프라이트

방향별 × 상태별 프레임을 전부 그려 하나의 스프라이트시트로 합침.

```
방향 (symmetric 기준, 4방향 제작 + 좌우 반전 = 8방향):
  Down(앞), Down-Right(대각 앞), Right(옆), Up(뒤)
  → Left, Up-Left, Down-Left, Up-Right는 반전으로 자동 생성

방향 (asymmetric 기준):
  8방향 전부 제작. Up/Down이 대칭이면 6방향.
```

| 상태 | 방향 (sym) | 방향 (asym) | 프레임 | 설명 |
|:---|:---|:---|:---|:---|
| Idle | 4 | 8(6) | 3 | 대기 루프. 호흡, 부품 진동 등 미세 움직임 |
| Walk | 4 | 8(6) | 6 | 이동 사이클. MS에 따라 재생 속도 조절 |
| Attack | 4 | 8(6) | 5 | 백스윙→타격(hit_frame)→후딜→복귀 |
| Death | 1 | 1 | 5 | 피격→흔들→붕괴→쓰러짐→잔해. 비루프 |

```
프레임 수 합계:
  symmetric:   4×3 + 4×6 + 4×5 + 1×5 = 61장
  asymmetric:  6×3 + 6×6 + 6×5 + 1×5 = 89장 (Up/Down 대칭)
               8×3 + 8×6 + 8×5 + 1×5 = 117장 (완전 비대칭)
```

### 부속 에셋

```
그림자:
  유닛 아래 반투명 타원. 크기 클래스별 1장 (전체 공유).
  소형 12×6px / 중형 20×10px / 대형 32×16px
  → 유닛별 제작 불필요.

HP 바:
  코드 렌더. 스프라이트 아님.
  아군 초록 / 적 빨강 / 배경 검정반투명.

★등급 비주얼:
  팔레트 스왑 (셰이더 or 팔레트 테이블).
  기본 스프라이트에 "교체 가능 색상 영역"을 지정색으로 칠해둠.
  ★1 기본 / ★2 은색·청색 하이라이트 / ★3 금색·발광
  → 별도 스프라이트 불필요.
  선택적으로 ★2/★3에 발광 테두리 오버레이(등급별 공용 1~2장).
```

### 투사체 (원거리 유닛만, 무기 타입별 공유)

```
유닛별이 아닌 "무기 태그별" 공유 에셋.

구성: 비행 2~3프레임 + 적중 이펙트 2~3프레임 = 4~6장/타입

스팀펑크 예:
  #화기 (사수, 터렛):    황동 총알    ●→ ●→ 💥 ✦     4장
  #전기 (드론, 정찰기):  전기 볼     ◉→ ◎→ ○→ 💥 ✦ · 6장

→ 테마당 2~3가지 투사체 타입. 5테마 × 2~3 × 5장 ≈ 50장.
```

### 공용 이펙트 (전체 게임 공유)

```
피격 스파크 (근접):     3프레임
피격 불꽃 (원거리):     3프레임
버프 글로우 (개량/충전): 4프레임 — 유닛에 오버레이
제조 출현 (유닛 생성):   4프레임 — 팝인 효과
사망 파편:              크기별 3종 × 3프레임 = 9프레임
체인 연결선:            2~3프레임 — 트리거 발동 시 카드 간 연결

합계: ~25장
```

### 유닛 1종 완성 구성 예 (태엽 거미, symmetric 소형 근접)

```
[본체 스프라이트시트]
  Idle 4방향×3f, Walk 4방향×6f, Attack 4방향×5f, Death 1방향×5f
  → 61장 (32×32px) → 스프라이트시트 1장 (256×256px)

[팔레트 스왑] ★1 기본 / ★2 은색 / ★3 금색 → 추가 제작 0장

[공유 에셋 참조]
  그림자: shadow_small (공용)
  투사체: 없음 (근접)
  피격: spark_melee (공용)
  사망: debris_small (공용)

아티스트 실작업: 61장
```

## 기타 에셋

- **카드 아트**: 50장 일러스트 + ★등급별 프레임.
- **전장 배경**: 바닥 타일맵 or 배경 이미지. Range 기반 거리감 확보.
- **UI**: 상점, 벤치, 필드 슬롯, 업그레이드 슬롯, 체인 연출.

## 성능 최적화

400유닛에서 진짜 병목은 렌더가 아니라 **게임 로직**(AI·타겟팅·데미지).
"유닛 수가 많을수록 그리는 비용보다 판단하는 비용이 더 크다." (SC2)

### 1. 시뮬레이션/렌더 분리 [필수] — SC2

```
게임 틱 (20fps):
  400유닛 AI 판단 + 타겟팅 + 데미지 계산 + 위치 갱신
  버짓: 50ms (여유로움)

렌더 프레임 (60fps):
  이전 틱 → 현재 틱 위치를 선형 보간(lerp)
  로직 없음, 보간+렌더만
```

CPU 집약 연산 초당 60회 → 20회. 게임 로직 CPU **3배 절감**.

### 2. 부대 단위 AI [필수] — Total War

개별 유닛이 아닌 부대(카드) 단위로 판단. 카드=부대 구조에 자연스럽게 매핑.

```
개별 AI:  유닛 400기 × 판단 = 400회 AI 틱
부대 AI:  카드 최대 8장(아군) × 판단 = 8회 AI 틱
          개별 유닛 = 부대 명령 + 포메이션 오프셋

→ AI 비용 ~50배 절감
```

### 3. 오브젝트 풀링 [필수] — Vampire Survivors

유닛 생성/사망이 빈번(제조, 전투 사망). new/delete 반복 = GC 스파이크.

```
게임 시작 시 유닛 객체를 풀에 미리 할당 (최대 전투 규모 기준)
사망 → 풀에 반환 (메모리 해제 안 함)
생성 → 풀에서 꺼내 초기화 (메모리 할당 안 함)

→ GC 0, 할당 비용 0. 성장 체인의 대량 제조에서 특히 중요.
```

### 4. 공간 분할 격자 [필수] — SC2, Brotato

타겟팅("가장 가까운 적 찾기") + 충돌 판정 최적화.

```
나이브:   400 × 400 = 160,000 거리 비교
공간 격자: 자기 셀 + 인접 셀만 = 수십 번 비교
```

O(N²) → O(N). 구현 단순, 효과 큼.

### 5. 데이터 지향 배열 구조 [권장] — Factorio

같은 필드끼리 연속 배열로 관리. CPU 캐시 효율 극대화.

```
OOP:  units = [{ pos, hp, atk, state }, { pos, hp, atk, state }, ...]
      → 객체가 메모리에 흩어짐 → 캐시 미스 ~30%

DOD:  positions = [vec2, vec2, ...]
      hps       = [int, int, ...]
      atks      = [int, int, ...]
      → 같은 필드 연속 접근 → 캐시 미스 ~2%

→ 동일 연산 ~8배 차이. 유닛 수 늘수록 격차 확대.
```

### 6. Flow Field 경로탐색 [권장] — They Are Billions

근접 유닛 다수가 동시에 적진으로 이동할 때, 개별 A*는 비현실적.

```
Flow Field: 목적지 기준 전장 전체에 "흐름 방향" 1회 계산.
각 유닛은 자기 위치의 방향값만 읽음 = O(1).

400유닛이든 4000유닛이든 경로 비용 동일.
```

### 7. 원형 충돌 판정 [권장] — Brotato

사거리/교전 판정에 원형 충돌 사용. 폴리곤 교차 불필요.

```
distance² < (r1 + r2)²
→ 곱셈 3회, sqrt 불필요. 공간 격자와 조합하면 실질 비용 0에 수렴.
```

### 8. GPU 인스턴싱 [권장] — SC2

같은 유닛 타입 = 같은 텍스처 아틀라스, 같은 쿼드 메시.

```
인스턴스 데이터 (유닛당 32 bytes):
  position:  vec2 (8B)
  frame_uv:  vec2 (8B)    ← 현재 프레임 UV 오프셋
  tint:      vec4 (16B)   ← 팀 컬러, ★등급

태엽 거미 120기 = 1 드로우콜 + 3,840 bytes
```

### 9. 화면 밖 컬링 [권장] — SC2

200v200이면 화면에 동시에 보이는 건 50~100유닛 수준.
나머지는 게임 로직만 실행, 렌더 스킵.

### 10. 애니메이션 업데이트 주기 분리 [선택] — SC2

```
화면 중앙:     매 프레임 업데이트
화면 가장자리:  2프레임마다
화면 밖:       정지 (상태만 유지)
```

### 11. 더티 렉트 렌더링 [선택] — AoE2

변화가 있는 영역만 갱신. 빌드 페이즈(유닛 정적)에서 체인 이펙트 영역만 다시 그릴 때 유용.

## 아트 파이프라인

```
3D 모델링 (참조용)
  ↓
4방향 프레임 렌더 → 스프라이트 시트
  ↓
텍스처 아틀라스 패킹 (유닛 타입별 or 테마별)
  ↓
팔레트 스왑 생성 (★등급별)
  ↓
모바일: ASTC/ETC2 압축
웹: PNG 아틀라스 (WebGL 로드)
```

유닛 40종 기준 제작 규모:
- 3D→2D 프리렌더 파이프라인: 유닛당 2~3일 → 전체 3~4개월 (1인)
- 순수 2D 픽셀아트: 유닛당 3~5일 → 전체 4~6개월 (1인)
