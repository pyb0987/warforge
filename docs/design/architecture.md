# 2층 구조: 유닛 풀 & 카드 설계 (핵심 아키텍처)

## 설계 원칙

> **유닛과 카드는 별개로 존재한다.**
> - 유닛: 공유 데이터베이스 (어떤 카드에든 포함 가능)
> - 카드: 유닛 조합 + 트리거 효과
> - 효과: 유닛 타입을 참조하여 카드 간 시너지 발생

## 유닛 풀 (공유 데이터베이스)

```
총 50종 (테마별 10종 + 중립 10종). 상세는 테마별 파일 참조:
  units-steampunk.md  10종 (#기계, 균형, 평균 ATK 4.2)
  units-druid.md      10종 (#생체, 소수 정예, 평균 ATK 7.5)
  units-predator.md   10종 (#생체, 물량, 평균 ATK 3.3)
  units-military.md   10종 (#생체+#기계, 정예, 평균 ATK 5.9)
  units-neutral.md    10종 (#기계/#생체 50:50, 평균 ATK 5.7)
```

## 유닛 태그 vs 카드 태그

```
유닛 태그: 유닛 DB에서 부여. 유닛당 2~3개. 카드 효과의 대상 지정에 사용.
  예: 증기 인형 → #기계 #증기

카드 태그: 카드 자체에 부여. 카드당 1~3개. 유닛 태그와 별도.
  예: [증기 조립소] → #스팀펑크 #생산
      (구성 유닛의 태그 합집합이 아님!)

참조 구분:
  유닛 태그 참조: "아군 카드 내 #기어 유닛 전체 개량"
  카드 태그 참조: "#생산 카드가 필드에 3장 이상이면 ~"
```

## 2층 구조의 장점

| 장점 | 설명 |
|------|------|
| **유닛 풀 깊이** | 유닛 타입마다 고유한 역할/스탯/태그 |
| **카드 간 시너지** | 카드 효과가 유닛 타입을 참조 → 다른 카드의 유닛도 영향 받음 |
| **전투 깊이** | 각 유닛 개별 전투 + 유닛별 특수능력 |
| **빌드 다양성** | 같은 유닛 타입을 여러 카드에서 활용하는 조합 가능 |

## 유닛 설계 범위

| 구분 | 범위 | 비고 |
|------|------|------|
| 공격력 | 1~20 | 유닛 개별 수치 |
| 체력 | 20~500 | 유닛 개별 수치 |
| 유닛 수/카드 | 1~10 | 카드당 구성 유닛 수 |
| 스탯 산출 | 10~2000 | 카드당 총합 (유닛 수 × 평균 스탯) |

## 태그 체계 (증기기관 테마 예시)

| 카테고리 | 태그 | 용도 |
|----------|------|------|
| 부품 | #기어, #스프링, #튜브, #보일러 | 동일 부품 시너지 |
| 동력 | #증서, #태양, #마력, #디젤 | 동력 관련 효과 |
| 기계유형 | #자동인형, #증기기사, #골렘, #장치 | 유닛 분류 |
| 상태 | #정상, #과열, #녹슨 | 상태依存 효과 |

## 카드 구성 요소

```
[부대명] ★레어리티
- 유닛 수: N명
- 스탯: 공격력 / 체력 (유닛당)
- 트리거 조건: 어떤 이벤트(기본/키워드)에 반응하는가 + 위치 조건
- 효과: 무엇을 하는가
- 출력 이벤트: 효과가 어떤 이벤트를 생성하는가
- 발동 상한: 라운드당 최대 반응 횟수
- 카드 태그: 1~3개 (유닛 태그와 별도)
- 업그레이드 슬롯: 5개
- 성장 누적: 카드 효과에 의한 스탯 modifier 누적 (아래 참조)
```

## 카드 레벨 스탯 modifier

> **강화(개량/성장/변태/훈련)는 유닛이 아닌 카드에 누적된다.**
> 기본은 카드의 모든 유닛에 적용. 태그 지정 시 해당 태그 유닛만 적용 (예: #태엽 ATK +5%).
> 유닛이 카드를 떠나면 기본 스탯으로 복귀.

### 스탯 계산 공식

```
유닛 실제 스탯 = 기본 × (1 + 성장 누적%) × (1 + 업그레이드%) × 곱연산
                 ─────   ──────────────   ──────────────────   ────────
                 유닛 DB   Layer 1: 유기적    Layer 2: 장비         Layer 3: 드문
                          (카드 효과 누적)    (테라진 구매)         (★3/보스/임계점)

레이어 내부: 가산 (성장 +5% + 성장 +3% = 성장 +8%)
레이어 간:  곱연산 (성장 layer × 업그레이드 layer)
Layer 3:   명시적 곱연산 효과 (★3, 보스 보상 등 드문 상황에만)
```

태그별 성장도 Layer 1에 합산: [전체 ATK +30%] + [#수목 ATK +8%] = #수목 유닛은 +38%

### modifier 소스

```
Layer 1 — 성장 누적 (슬롯 제한 없음):
  카드 효과(개량/성장/변태/훈련)에 의해 누적. 카드에 고정. 이동 불가.
  전체 적용: [전체 ATK +30%]
  태그별 적용: [#수목 ATK +8%]  [#태엽 ATK +12%]
  비용: 시간 (체류, 체인 반복)
  역할: "업그레이드의 승수를 높인다"

Layer 2 — 업그레이드 (5슬롯):
  테라진으로 구매. 카드에 영구 부착. 분리/이동 가능.
  예: [레어: ATK +10%]  [에픽: #기계 한정 ATK +15%]
  비용: 자원 (테라진)
  역할: "성장의 승수를 높인다"

두 레이어의 관계:
  레이어 간 곱연산 → 둘 다 투자할수록 서로의 가치가 올라감 (보완재)
  "성장이 높은 카드에 업그레이드를 붙이면 곱해져서 폭발한다"
```

### 표시 예시

```
[뿌리깊은 자] ★1                          체류: 5R
유닛: 고대 나무정령 ×3, 뿌리 수호자 ×2

┌─ 업그레이드 (5슬롯) ─────────────────┐
│ [레어: ATK +10%]  [에픽: HP +15%]  [ ] [ ] [ ] │
└──────────────────────────────────────┘
┌─ 성장 누적 ──────────────────────────┐
│ 전체 ATK +30%  │  전체 HP +15%       │
└──────────────────────────────────────┘

→ 고대 나무정령 실제 ATK: 6 × (1 + 0.30) × (1 + 0.10) = 6 × 1.30 × 1.10 = 8.58
→ 뿌리 수호자 실제 ATK:  5 × (1 + 0.30) × (1 + 0.10) = 5 × 1.30 × 1.10 = 7.15
```

### 유닛 이동 규칙

```
유닛이 카드를 떠날 때:
  → 기본 스탯으로 복귀 (원 카드의 성장 누적을 잃음)
  → 새 카드의 modifier를 즉시 받음

의미:
  "유닛은 카드에 뿌리내려 있다."
  이동은 가능하지만 성장분 소실이라는 대가가 있음.
  → 드루이드: 이동 비용이 높음 (체류 성장 소실)
  → 스팀펑크: 병기창에 유닛 집중 시, 원 카드의 개량분 소실 → 병기창의 성장분 적용
```

### 전 테마 적용

```
스팀펑크 개량: "대상 카드 내 #태엽 유닛 전체 개량(ATK +5%)"
  → 대상 카드의 성장 누적에 [#태엽 ATK +5%] 추가

드루이드 성장: "이 카드의 유닛 전체 성장(ATK +6%)"
  → 자기 카드의 성장 누적에 [전체 ATK +6%] 추가

포식종 변태: 유닛 자체를 교체 (약한 유닛 → 강한 유닛)
  → 카드의 성장 누적은 유지. 새 유닛도 modifier 적용.

군대 훈련: "유닛 수 비례 전체 강화"
  → 동적 계산. 성장 누적이 아닌 조건부 효과로 별도 처리.
```

## 합성 업그레이드 (★ 상승) — ★3에서 질적 변환

같은 카드 3장 → ★ 상승. ★1→★2: 양적 성장 (숫자 증가). ★2→★3: **질적 변환** (새로운 능력).

### ★2 설계 지침 (시뮬레이션 검증 완료)

```
핵심 원칙: "발동 횟수가 아닌 발동당 효과를 강화한다"

발동 상한 증가(2/R→3/R)는 체인 길이에 의존하여 효과가 불안정.
효과 자체를 강화하면 체인 길이와 무관하게 일정한 파워 증가.

★2 변화 유형 (1개 선택):
  ① 양 2배     스폰 1기→2기, 개량 5%→7.5~10%     ← 가장 체감 큼
  ② 범위 확대   인접→양쪽, self→adj 추가            ← 체인 파급 효과
  ③ 태그 확대   #태엽→#태엽+#전기                   ← 개량 카드에 적합
  ④ 수치 강화   버프 10%→15%, 골드 2→3              ← 비체인 카드에 적합
  ⑤ 효과 추가   기존 효과에 부가 효과                  ← 드물게 사용

금지:
  ✗ 발동 상한만 +1 (체감 미미, 시뮬 검증됨)
  ✗ 새로운 메커니즘 추가 (★3의 영역)

★3 효과 유형:
  ★3은 질적 변환. 기존에 없던 새로운 차원의 효과.
  - 패시브/오라: 카드 존재 시 상시 적용 (트리거 불필요)
    예: "자신의 #증기 유닛 공격속도 2배" (시한 보일러 ★3)
  - 브릿지: 다른 이벤트 타입을 생성 (체인 경로 확장)
    예: "개량 시 인접 카드 1기 제조" (태엽 공방 ★3)
  - 복합: 기존 효과 강화 + 새로운 축 추가

★2 총 파워 예산:
  효과 변경: 해당 카드 기여도 +20~40%
  무료 레어: 해당 카드 스탯 ×1.30
  합산: 해당 카드 총 기여 약 ×1.5~1.8

★2→★3 역할 분리:
  ★1: 기본 효과
  ★2: 같은 효과, 수치 ×1.5~2  ← "더 잘한다"
  ★3: 새로운 효과 유형 추가     ← "다른 존재가 된다"

  ★2 원칙: 카드의 정체성을 바꾸지 않고 기존 역할을 강화.
           핵심 수치 직접 강화 (5%→7.5%, 1기→2기).
           범위 확대, 부가 효과 추가는 ★3의 영역.

  ★2 티어별 예외:
    T1~T3: 원칙대로. 수치 강화만.
    T4~T5: 수치 변경이 질적 차이를 만드는 설계 허용.
           예: 사거리 임계점(12기→8기당 R+1) = 같은 메커니즘이지만 도달 가능한
               사거리가 달라져 전투 양상 자체가 변함.
           ★2 달성이 어려우므로 (T4 3장, T5 3장) 보상의 질이 높아야 함.

  ★3 원칙: 카드의 역할이 질적으로 변모. "같은 카드인데 다른 느낌".
           - 패시브/오라 (카드 존재 시 상시 적용)
           - 브릿지 (새 이벤트 타입 생성, 체인 경로 확장)
           - 범위 확대 (self→양쪽, 단일→전체)
           - 1회 임계점보다 상시 효과 선호
```

### 카드 효과 설계 원칙 (전 테마 공통)

```
1. 보상은 보여야 한다
   ✗ "ATK +16%" → 보이지 않는 숫자. 체감 불가.
   ✓ "카운터 10 → 1 테라진" → 쌓이는 게 보이고, 보상이 구체적.
   중간 이벤트(체인 링크)의 스탯 버프는 OK.
   최종 보상은 유닛 추가/자원 획득/업그레이드 등 구체적이어야 함.

2. 효과는 의사결정을 만들어야 한다
   ✗ 자동 발동 + 자동 적용 → 플레이어 관여 없음.
   ✓ "판매 시 유닛 흡수" → "이 카드를 언제 팔까?"
   ✓ "레어 업그레이드 3택1" → 선택의 순간.

3. 같은 축의 중복을 피한다
   ✗ ATK 개량 카드 3장 → 같은 효과의 양적 차이만.
   ✓ ATK 개량 + 방어막 + 자원 변환 → 다른 축.
   같은 트리거에 반응하더라도 효과의 축이 달라야 함.

4. 트리거 주체를 명확히 한다
   ✗ "제조 시" → 자기 루프 위험.
   ✓ "다른 카드의 제조 효과 발동 시" → source 기준 명확.
   자기 효과 자기 반응 = 원천 차단.

5. 효과 태그를 구분한다
   [고유효과]: 같은 카드 2장+ → 왼쪽 1장만 적용. 강력 효과 중복 방지.
   [지속 효과]: 카드 존재 시 상시 적용. ★3 질적 변환에 사용.
   (체인 미발생): 이 효과가 동종 이벤트를 재생성하지 않음. 체인 종결.

6. T5 capstone은 게임 시스템과 연결한다
   T1~T4: 체인 내부 효과 (제조/개량/전투).
   T5: 게임 시스템(상점/판매/업그레이드)과 연결 → 독보적 가치.

7. T4 선택지는 다른 축이어야 한다
   같은 T4 2장 중 선택 → 축이 달라야 진짜 선택.
   ✗ 공격 A vs 더 강한 공격 B → 양적 차이.
   ✓ 화력/사거리 vs 경제/업그레이드 → 다른 축.
```

## 트리거 효과 설계 원칙

> 성장 체인(주력): 유닛 수 증가, 스탯 영구 강화 → 군대를 키우는 효과.
> 전투 체인(보조): 일시적 버프, 단순 강화 → 전투를 살짝 보조하는 효과.
> **⚔공격은 트리거 조건만 가능, 효과로 생성 불가.**
