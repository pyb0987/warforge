# 전투 시스템 & 적 파워 곡선

## 리얼타임 자동 전투 (SC1 스타일)

- SC1 스타일 군집 전투. **전투는 절대 멈추지 않는다.**
- 적 군대는 단순 스탯 기반 (트리거/체인 없음)
- **전투력 = 성장 체인으로 키워진 군대의 누적 결과**

## 유닛 전투 스탯

```
유닛 풀 스탯: ATK, HP, AS(공속), Range(사거리), MS(이동속도), 태그

공격 속도 (AS) — 공격 간 간격:
  빠름 0.5초 / 보통 1.0초 / 느림 1.5초
  → DPS = ATK / AS. 빠른 약타 vs 느린 강타.

사거리 (Range) — 공격 가능 거리:
  근접 0 / 근거리 2 / 원거리 4 / 장거리 6
  → 원거리 유닛은 제자리에서 공격 시작. 근접이 도달 전에 딜.

이동 속도 (MS) — 전투 중 이동:
  빠름 3 / 보통 2 / 느림 1
  → 초기 배치에 영향 없음 (진영 유지). 교전 중에만 적용.
  → 근접 유닛은 MS로 적에게 접근. 빠를수록 빨리 교전 시작.
  → 원거리 유닛은 사거리 안이면 이동 없이 공격.

방어력 (DEF) — 받는 피해 감소:
  기본 0 (모든 유닛). 업그레이드로만 부여 가능.
  → 데미지 = ATK - DEF (최소 1)
```

## 전투 흐름

```
1. 양쪽 유닛이 진영에 배치 (진형 유지, 이동 없음)
2. 사거리 안에 적이 있는 유닛은 즉시 공격 시작
3. 사거리 밖인 유닛(주로 근접)은 MS로 적을 향해 이동
4. 사거리 안에 들어오면 공격 시작
5. AS 간격으로 반복 공격. 타겟: 가장 가까운 적.
6. 데미지: ATK - DEF (최소 1). HP 직접 차감.
7. HP 0 = 사망 (💀 이벤트). 한쪽 전멸 시 종료.
8. 타임아웃 60초.

전투 역할 (스탯 조합):
  스웜:   낮은ATK, 낮은HP, 빠른AS, 근접, 빠른MS → 물량 러시
  탱커:   중간ATK, 높은HP, 느린AS, 근접, 느린MS → 전선 유지
  딜러:   높은ATK, 낮은HP, 보통AS, 원거리       → 후방 딜
  포격:   매우높은ATK, 낮은HP, 느린AS, 장거리    → 일방적 포격
```

## 업그레이드 전투 효과

```
공속:       "공속 +20%" → AS 1.0 → 0.8. DPS 25% 증가.
사거리:     "사거리 +2" → 근접→근거리, 원거리→장거리.
스플래시:   "공격 시 주변 적 50% 피해"
라이프스틸: "피해의 25%만큼 HP 회복"
방어력:     "방어력 +3" → 받는 피해 -3 (최소 1)
관통:       "방어력 무시" → DEF 무시하고 ATK 전체 적용
```

## 전투력 표시 (Combat Power)

```
공식:
  유닛 CP = (ATK / AS) × HP
  카드 CP = Σ 유닛 CP (해당 카드의 모든 유닛)
  전체 CP = Σ 카드 CP

의미:
  ATK / AS = DPS (초당 데미지). 공속이 빠를수록 DPS 높음.
  DPS × HP = 그 유닛이 생존 동안 총 기여할 수 있는 전투 가치.

예시:
  태엽 거미 (ATK 2, AS 0.5, HP 20):  CP = 4 × 20 = 80
  증기 타이탄 (ATK 4, AS 1.5, HP 100): CP = 2.7 × 100 = 267
  광란의 야수 (ATK 8, AS 0.5, HP 35):  CP = 16 × 35 = 560

용도:
  - 전투 전 양 진영 CP 비교 표시 (플레이어 vs 적)
  - 성장 체인 발동 후 CP 변화량 실시간 표시 ("체인으로 +1,200 CP!")
  - 라운드 결과 화면에 누적 CP 그래프
  - 스크린샷/공유용 숫자 ("R12에서 CP 50,000 달성!")
```

## 전투 중 체인 (보조)

- 전투 중 발생하는 이벤트(💀사망 등)에 반응하는 단순 체인
- 주로 1~2링크의 짧은 체인 (사망→버프 정도)
- 빌드의 보조 수단이지, 전투를 뒤집는 주력이 아님

## 적 파워 곡선 (목표 — ★2+업그레이드 기준)

```
★2 + 업그레이드 빌드로 클리어 가능. ★3은 오버킬(사기의 쾌감).

감정 아크:
  R1~R3:   플레이어 > 적       "배우는 구간, 여유. 보드 채우기."
  R4:      ★보스. 기본 체인 필요.     → "기반" 보상
  R5~R7:   플레이어 ≈ 적       "빌드가 맞물려야 이긴다." ★2 시작.
  R8:      ★보스. ★2 없으면 패배.    → "전환" 보상
  R9~R11:  플레이어 < 적       ★2 + 업그레이드 필요. 빌드 최적화.
  R12:     ★보스. ★2 완성 빌드 시험.  → "완성" 보상
  R13~R14: 적 더 강화. ★2 + 업그레이드 풀빌드로 빡빡하게 돌파.
  R15:     ★최종 보스. 완성된 빌드만 돌파 가능. ★3 있으면 압도.

→ 적이 불합리하게 강해야 업그레이드와 체인이 "사기"가 된다.
→ ★3은 클리어 필수가 아닌 "오버킬" → 달성 시 극적 쾌감.
```

## 적 구성 프레임워크 (프리셋 + 랜덤 업그레이드)

```
구조:
  라운드별 프리셋 풀 3~4개 중 1개 랜덤 선택.
  프리셋 = 유닛 구성 (역할 비율 + 스탯) + 랜덤 업그레이드.
  같은 라운드라도 적 구성이 다르므로 런 간 다양성 확보.

프리셋 유형 (역할 분포):
  "스웜 러시"    소형 다수, 빠른 AS, 낮은 HP        → 물량으로 압도
  "중장 벽"      대형 소수, 높은 HP/DEF, 느린 AS    → 전선 유지, 돌파력 필요
  "저격 편대"    원거리 다수, 높은 ATK, 낮은 HP      → 빠른 접근 또는 사거리 필요
  "균형 혼합"    탱커+딜러+원거리 혼합               → 범용 대응 필요

랜덤 업그레이드:
  R1~R3:   업그레이드 없음 (순수 스탯)
  R4~R7:   0~1개 커먼 업그레이드 랜덤 부여
  R8~R11:  1~2개 커먼/레어 업그레이드 랜덤 부여
  R12~R14: 2~3개 레어 업그레이드 랜덤 부여
  R15:     3개+ 레어/에픽 업그레이드 보유

  → 플레이어가 사용하는 것과 같은 업그레이드 풀에서 부여
  → "이 적은 스플래시가 있다 → 유닛 분산 배치" 같은 적응형 플레이 유도
  → 런마다 적 업그레이드 조합이 다르므로 리플레이성 ↑

프리셋 풀 규모:
  일반 라운드: 라운드당 3~4개 프리셋
  보스 라운드: 전용 프리셋 1개 (고정, 특성 보유)
  총: 일반 11R × 3~4 + 보스 4 = 37~48개 프리셋
  → 카드 풀 확정 후 설계
```

## 보스 설계 (특성 보스)

```
보스는 일반 적과 질적으로 다르다. 각 보스에 고유 특성 1개.

R4★ "기반 보스" — 스웜 특성
  유닛 수 1.5배, ATK 약함. 물량으로 플레이어 전선 시험.
  → 체인이 작동하는지 검증하는 최초의 벽.

R8★ "전환 보스" — 중장 특성
  유닛 수 적음, DEF 보유, HP 높음. 순수 물량으로 돌파 불가.
  → ★2 + 관통/높은 ATK가 없으면 시간 내 처치 불가.

R12★ "완성 보스" — 속공 특성
  AS 빠름, 원거리 다수, 높은 ATK. 플레이어 유닛이 빠르게 녹음.
  → ★2 + 업그레이드 풀빌드(HP/DEF/라이프스틸)가 필수.

R15★ "최종 보스" — 복합 특성
  모든 역할 혼합 + 스탯 극강 + 레어/에픽 업그레이드 보유.
  → ★2+업그레이드 풀빌드로 빡빡하게 돌파. ★3 있으면 압도.

보스 특성은 프로토타입 후 조정. 핵심: 각 보스가 다른 빌드 축을 시험.
```

## 스케일링 방법론

```
적 총 파워(R) = 기본값 + R × 선형계수 + R² × 2차계수

플레이어 파워가 2차 성장(유닛추가×강화)이므로 적도 2차 스케일링.
라운드 구간별 2차계수 조절로 감정 아크 구현:

  R1~R3:   2차계수 낮음   (플레이어 > 적)
  R4~R7:   2차계수 중간   (플레이어 ≈ 적)
  R8~R12:  2차계수 높음   (플레이어 < 적, ★2+업글 필요)
  R13~R15: 2차계수 매우높음 (풀빌드 필요)
  보스:    해당 구간의 1.5~2배

구체적 계수 도출 절차 (카드 풀 확정 후):
  1. 대표 빌드 3~4개로 ★1/★2/★3별 라운드별 파워 워크스루
  2. "★2+업글 빌드가 R15를 간신히 돌파"하는 적 파워 역산
  3. 보스 = 해당 구간 일반 적의 1.5~2배로 설정
  4. 시뮬레이션으로 클리어율 검증 (목표: 30~40%)
```
