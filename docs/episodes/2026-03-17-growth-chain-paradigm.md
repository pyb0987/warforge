# 2026-03-17: 성장 체인 패러다임 확립 + 2층 유닛/카드 아키텍처

## Topic 1: 체인 = 성장 메카닉 (핵심 패러다임 전환)

### Process
- 초기 문서: 체인을 "전투 중 발동하는 연쇄 메카닉"으로 설계
- 유저 피드백: "체인은 군대를 키우는 것이 주력. 전투 중 체인은 보조."
- 예시: "옆 카드 유닛수 증가 시 → 유닛 1기 배양", "전투 패배 시 → 유닛 복사"
- 결론: 성장 체인(빌드 페이즈) = 주력, 전투 체인 = Option B(사망→버프 정도)
- 문서 전면 재구성: 성장 체인 시스템을 독립 핵심 섹션으로 승격

### Negative Knowledge
- 체인을 전투 메카닉으로 해석하면 유저의 핵심 재미("키워가는 맛")와 충돌
- "공격→공격→공격" 빌드: 공격은 효과로 생성 불가 규칙과 모순 → 제거됨

## Topic 2: 핵심 재미 프로필 확립

### Process
- 유저 레퍼런스: 스타선술집전투 아몬도전
- 핵심 쾌감: 불합리한 적 → 실력(조합+업그레이드)으로 "사기" → 압도
- 업그레이드가 "어렵지만 가능해야" 한다
- 시각 피드백(색상/반짝임)이 그 자체로 즐거움
- → ★3 질적 변환 원칙, 적 파워 곡선("불합리" 구간), 시각 피드백 원칙 도출

## Topic 3: 2층 유닛/카드 아키텍처 (유저 자체 추가)

### Process
- 유저가 직접 "유닛 풀 & 카드 설계" 섹션 추가
- 핵심 변경: 유닛과 카드를 별개로 분리. 카드 = 유닛 조합 + 트리거 효과
- 효과가 유닛 태그를 참조 → 카드 간 시너지 (다른 카드의 유닛도 영향)
- 증기기관 테마로 구체적 카드 10장 + 아키타입 3종 설계
- 기존 카드 구조(단일 유닛 × N)와 다른 새로운 모델

### 주의 사항
- 문서에 인코딩 깨짐 있음 (증기 inúmer, 自身, 1층첩가 등) — 정리 필요
- 확정된 방향 테이블의 "테마/세계관: 미정"과 카드풀의 증기기관 테마가 불일치
- 기존 예시 카드(양성사, 배양소 등)와 새 카드(증기 발생기, 기어 훈련소 등)가 병존 — 통합 필요
