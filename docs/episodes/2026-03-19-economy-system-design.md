# 2026-03-19: 경제 시스템 전면 설계 + 전투/카드 기반 확정

## Topic 1: 2화폐 경제 (골드 + 테라진)

### Process
- 목표: 경제 파라미터 확정 (레벨업, 전투 보상, 상점 확률, 업그레이드 상점, 보스 보상)
- BG(122g)와 비교 → 우리 게임(108g)이 부족 우려 → 레벨업 비용을 높이고 자동 할인 도입
- 상점 5단계 → 6단계 전환 (티어 분포 변화가 너무 급격)
- 업그레이드를 골드로 사면 → 골드 부적/커맨더가 모든 축을 지배 → 테라진 별도 화폐 도입
- 판매 손실 -1g → 전액 환급 (골드부족+이자+벤치 8칸이 신중함 담당)

### Key Decisions
- 레벨업: 5/7/8/11/13 (6단계), 자동 할인 -1g/R, 연속 레벨업 허용
- 테라진: 승리+2/패배+1, 업글 커먼4t/레어8t, 리롤1t
- 보스 보상: 풀9/4택1, 보스별 전용, 유형 2⚡4🔄2💥1📐, 규칙 변경 중심
- ★3: 드문 보너스 (3×3=9 유지). ★2+업글로 클리어 가능.
- 판매 전액 환급. 실질 예산 ~122g로 BG와 동등.

### Negative Knowledge
- 레벨업 4/5/6/7 (예산의 20%): 러쉬 레벨이 지배 전략이 될 수 있음 → 100%+ 수입 비율 필요
- 상점 5단계: 레벨업마다 분포 ~40% 변화 → 너무 급격. 6단계로 ~26%로 완화.
- 보스 보상에 "초반 보상 + 후반 보상" 혼합 (방안 C): 낮은 등급 = 가짜 선택지. 보스별 전용 풀(방안 B)이 올바름.
- 업그레이드 슬롯 1/2/3 체계: 슬롯 퍼즐보다 효과 선택이 핵심 → 1슬롯 통일.
- 판매 손실 -1g: "신중한 구매" 의도였으나 카드 교체/탐색/★2 방향전환을 과도하게 억제.

## Topic 2: 전투 시스템 확정 (SC1 스타일)

### Process
- 전투 시스템 구체화가 카드 설계의 전제 조건 (순서 수정: 전투→카드풀)
- SC1/선술집전투 레퍼런스 → 유닛 풀에 AS/Range/MS 추가 (방안 A)
- 초기 배치 시 진형 유지 (MS로 이동하지 않음), 교전 중에만 MS 적용
- DEF 기본 0, 업그레이드로만 부여

### Key Decisions
- 유닛 스탯: ATK, HP, AS(0.5/1.0/1.5), Range(0/2/4/6), MS(1/2/3)
- 타겟팅: 가장 가까운 적
- DEF: 기본 0, 업그레이드 전용. 데미지 = ATK - DEF (최소 1)
- 필드 6칸 시작, 최대 8, 커맨더/부적/보스로만 증가

## Topic 3: 카드 설계 기반 원칙

### Process
- T1 카드 7장 설계 → 키워드/트리거 시스템 논의로 발전
- ★2 합성 규칙: 기존 2장 유닛 합산 (성장분 포함), 3번째는 촉매
- 배양 대상: 효과 텍스트로 지정 (최다/최강/무작위)
- 트리거 언어: 키워드별 독립 + 결과 기반 상위 범주

### Negative Knowledge
- "복제/조립/수리 → 모두 배양 이벤트": 틀림. 키워드별 독립 트리거여야 함.
  "배양했을 때"는 "복제"에 반응하면 안 됨. (유희왕 원칙)
- "유닛이 추가될 때"는 상위 범주로 모든 키워드에 반응 OK.
- 스팀펑크 테마에서 "배양"은 생물적 느낌 → "복제/조립/수리" 등 기계적 표현 사용.
